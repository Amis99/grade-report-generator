1. 시작하기

1.1 로그인

본사 관리자는 메인 포털(index.html)을 통해 로그인합니다. 관리자 계정의 아이디와 비밀번호를 입력하여 접속합니다.

  1. 웹 브라우저에서 프로그램 URL에 접속합니다.
  2. 아이디(사용자명)와 비밀번호를 입력합니다.
  3. 로그인 버튼을 클릭합니다.
  4. 인증이 완료되면 대시보드로 이동합니다.

⚠️ 관리자 vs 기관 관리자
본사 관리자(admin)는 설정 페이지에서 관리자 메뉴(가입 승인, 사용자 관리)가 추가로 표시됩니다. 기관 관리자(org_admin)에게는 이 메뉴가 보이지 않습니다.

1.3 비밀번호 변경

설정 페이지 또는 사이드바 하단 사용자 메뉴에서 비밀번호를 변경할 수 있습니다.

  1. 사이드바에서 ⚙️ 설정을 클릭합니다.
  2. 비밀번호 변경 섹션에서 현재 비밀번호, 새 비밀번호, 새 비밀번호 확인을 입력합니다.
  3. 변경 버튼을 클릭합니다.

2. 대시보드

로그인 후 가장 먼저 표시되는 화면으로, 현재 상태를 한눈에 파악할 수 있습니다.

주요 구성

  • 환영 인사: 관리자 이름, 소속, 현재 날짜를 표시
  • 할 일 피드: 진행중인 과제 수, 답안 미입력 시험 수, 미제출 과제 등
  • 최근 활동 타임라인: 시험 생성, 학생 추가, 채점 활동 등 최근 이벤트

3. 시험 관리

시험의 생성, 수정, 삭제 및 문제 관리를 수행하는 핵심 기능입니다. 본사 관리자는 모든 기관의 시험을 조회하고 관리할 수 있습니다.

3.1 시험 목록

사이드바에서 📝 시험 관리를 클릭하면 시험 목록이 표시됩니다. 시행일 기준 최신순으로 정렬됩니다.

3.2 시험 생성

  1. 시험 관리 화면에서 새 시험 만들기 버튼을 클릭합니다.
  2. 시험명, 시행일, 설명을 입력합니다.
  3. 저장을 클릭하여 시험을 생성합니다.
  4. 생성된 시험을 클릭하여 문제를 추가합니다.

3.3 시험 수정 및 삭제

  • 수정: 시험 목록에서 시험을 선택한 후 시험명, 시행일, 설명을 변경할 수 있습니다.
  • 삭제: 시험 상세 화면에서 시험 삭제 버튼을 클릭합니다. 확인 대화상자가 표시됩니다.
🚨 주의
시험을 삭제하면 해당 시험의 모든 문제, 답안, 성적 데이터가 함께 삭제됩니다. 이 작업은 되돌릴 수 없습니다.

3.5 CSV 가져오기 및 내보내기

문제를 CSV 파일로 일괄 등록하거나 내보낼 수 있습니다.

CSV 가져오기

  1. 시험 상세 화면에서 CSV 가져오기 버튼을 클릭합니다.
  2. 준비된 CSV 파일을 선택합니다.
  3. 미리보기를 확인한 후 가져오기를 클릭합니다.
💡 CSV 파일 형식
번호,유형,배점,영역,하위영역,지문,정답
예시: 1,객관식,4,독서,사실적 이해,가,3
인코딩은 반드시 UTF-8을 사용하세요.

CSV 내보내기

시험 상세 화면에서 CSV 내보내기 버튼을 클릭하면 현재 문제 목록이 CSV 파일로 다운로드됩니다.

4. 답안 입력

학생들의 시험 답안을 입력하는 화면입니다. 스프레드시트 형식으로 한 화면에서 여러 학생의 답안을 입력할 수 있습니다.

4.1 답안 입력 방법

  1. 사이드바에서 ✏️ 답안 입력을 클릭합니다.
  2. 상단 드롭다운에서 시험을 선택합니다.
  3. 답안 시트에서 학생명 셀을 클릭하면 학생 선택 모달이 열립니다.
  4. 각 문제 번호 열에 해당 학생의 답안을 입력합니다.
    • 객관식: 1~5 숫자를 클릭하여 선택
    • 서술형: 답안 텍스트 입력 후 점수 부여
  5. 저장하기 버튼을 클릭하여 답안을 저장합니다.

4.2 CSV 일괄 업로드

여러 학생의 답안을 CSV 파일로 한번에 업로드할 수 있습니다.

💡 답안 CSV 형식

서술형 문제는 점수를 직접 입력합니다.

8. 학생 관리

학생 정보의 등록, 수정, 삭제 및 학생 계정 관리를 수행합니다. 본사 관리자는 모든 기관의 학생을 관리할 수 있습니다.

8.1 학생 목록

이름 검색, 수강반 필터링이 가능하며, 체크박스로 여러 학생을 선택하여 일괄 작업을 수행할 수 있습니다.

8.2 학생 추가, 수정, 삭제

학생 추가

  1. + 학생 추가 버튼을 클릭합니다.
  2. 이름, 학교, 학년, 반, 번호를 입력합니다.
  3. 저장을 클릭합니다.

학생 목록에서 ✏️ (편집) 아이콘을 클릭하면 학생 수정 모달이 열립니다.

학생을 삭제하면 해당 학생의 모든 시험 답안, 성적 데이터가 함께 삭제됩니다.

8.4 중복 학생 찾기 및 병합

동일한 학생이 중복 등록된 경우를 찾아 병합합니다.

  1. 중복 학생 찾기 버튼을 클릭합니다.
  2. 시스템이 이름, 학교, 학년 등을 기준으로 중복 가능성이 있는 학생을 표시합니다.
  3. 각 중복 그룹을 확인하고 병합을 클릭합니다.
  4. 병합된 학생은 모든 시험 기록을 유지합니다.

자동 병합

모두 자동 병합 버튼을 클릭하면 이름, 학교, 학년이 정확히 일치하는 학생들을 자동으로 병합합니다.

⚠️ 주의
자동 병합은 정확히 일치하는 경우에만 동작합니다. 병합 후에는 되돌릴 수 없으니 신중하게 사용하세요.

11.3 관리자 메뉴

아래 기능은 본사 관리자(admin) 역할에게만 표시됩니다. 기관 관리자(org_admin) 및 학생(student) 역할에서는 접근할 수 없습니다.

11.3.1 가입 승인 대기

  1. 설정 > 관리자 메뉴를 엽니다.
  2. 가입 승인 대기 섹션에서 대기 중인 요청을 확인합니다.
  3. 각 요청의 사용자명, 이메일, 소속 기관을 검토합니다.
  4. 승인 또는 거절 버튼을 클릭합니다.
  5. 승인된 사용자에게는 org_admin 역할이 부여됩니다.

12. 부록

12.1 자주 묻는 질문 (FAQ)

Q: 기관 관리자와 본사 관리자의 차이점은 무엇인가요?
A: 본사 관리자(admin)는 모든 기관의 데이터를 조회하고 관리할 수 있으며, 가입 승인과 사용자 관리 기능을 사용할 수 있습니다. 기관 관리자(org_admin)는 자신이 소속된 기관의 데이터만 관리할 수 있습니다.
Q: 가입 승인을 하면 어떤 역할이 부여되나요?
A: 기본적으로 org_admin(기관 관리자) 역할이 부여됩니다. 승인 후 필요에 따라 설정 > 관리자 메뉴 > 사용자 관리에서 역할을 변경할 수 있습니다.
Q: 삭제한 데이터를 복구할 수 있나요?
A: 이전에 생성한 백업 파일이 있으면 설정 > 데이터 복원을 통해 복구할 수 있습니다. 백업이 없는 경우 복구가 불가능하므로 정기적인 백업을 권장합니다.
Q: CSV 파일 형식은 어떻게 되나요?
A: UTF-8 인코딩의 CSV 파일을 사용합니다. 첫 행은 반드시 헤더(열 이름)여야 합니다. Excel에서 저장 시 "CSV UTF-8 (쉼표로 분리)"를 선택하세요.
Q: 유사도 검사는 어떻게 작동하나요?
A: Amazon Bedrock AI 서비스를 사용하여 학생 제출 이미지를 분석합니다. 이미지의 내용과 구조를 비교하여 유사도를 퍼센트로 산출합니다. 80% 이상이면 의심 대상으로 분류됩니다.
Q: 학생 계정 비밀번호를 잊었을 때 어떻게 하나요?
A: 설정 > 관리자 메뉴 > 사용자 관리에서 해당 학생 계정의 "PW 초기화" 버튼을 클릭하여 임시 비밀번호로 재설정할 수 있습니다. 재설정된 임시 비밀번호를 학생에게 전달하세요.

12.2 문제 해결

문제원인해결 방법
로그인 실패 잘못된 아이디/비밀번호, 비활성화된 계정, 미승인 계정 아이디와 비밀번호를 확인하세요. 계정이 승인/활성화되어 있는지 관리자에게 문의하세요.
데이터 로딩 오류 네트워크 연결 문제, 서버 오류 인터넷 연결을 확인하고 페이지를 새로고침하세요. 문제가 지속되면 브라우저 캐시를 삭제하세요.
CSV 업로드 실패 잘못된 인코딩 (EUC-KR 등), 잘못된 형식 CSV 파일이 UTF-8로 인코딩되어 있는지 확인하세요. 헤더 행의 열 이름이 올바른지 확인하세요.
PDF 생성 오류 데이터 부족, 브라우저 호환성 시험 데이터와 답안이 모두 입력되어 있는지 확인하세요. Chrome 브라우저 사용을 권장합니다.
유사도 검사 실패 AI 서비스 연결 오류, 이미지 크기 초과 잠시 후 다시 시도하세요. 학생 제출 이미지가 너무 큰 경우 자동 압축이 적용됩니다.
💡 추가 도움
